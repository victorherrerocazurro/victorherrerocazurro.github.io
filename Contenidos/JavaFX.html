1. Introduccion

Framework para el desarrollo de aplicaciones de escritorio, embebidas y de movilidad.

Forma parte de la JDK 8.

Ofrece APIs como:

  • Graficos 2D y 3D.

  • UI controls, layouts y charts

  • Soporte para Audio y video

  • Efectos y animaciones

  • Soporte para HTML5 y CSS

Proporciona alto rendimiento al emplear motores nativos de renderizado, como Direct3D para Windows y OpenGL para Linux.

Tiene compatibilidad con APIs como Swing.

Emplea un lenguaje de marcado FXML para crear las vistas, equivalente a los HMTL del mundo web.

2. Instalacion

Para trabajar con JavaFX se necesita

  • JDK 8, que incluye el SDK de JavaFX, que se puede descargar de aquí

  • IDE con soporte para JavaFX: Eclipse, Netbeans, Intellij, etc

  • JavaFX Scene Builder: es una herramienta proporcionada por Oracle, que actualmente está en la version 2.0, para el diseño de ventanas JavaFX. Se puede descargar de aquí.

  • Se dispone de una distribucion de eclipse, con los plugin necesarios instalados aquí

3. Soporte Maven

Los proyectos pueden ser creados empleando las plantillas de los IDE o bien como proyectos Maven.

Para el desarrollo no se necesita nada especial, ya que es un proyecto java 8 normal.

Para la paquetizacion y el despliegue, se dispone de un plugin, da documentacion se puede obteer aquí.

<!-- https://mvnrepository.com/artifact/com.zenjava/javafx-maven-plugin -->
<build>
    <plugins>
        <!-- https://mvnrepository.com/artifact/com.zenjava/javafx-maven-plugin -->
        <plugin>
            <groupId>com.zenjava</groupId>
            <artifactId>javafx-maven-plugin</artifactId>
            <version>8.8.3</version>
            <configuration>
                <mainClass>sample.Main</mainClass>
            </configuration>
        </plugin>
    </plugins>
</build>

El cual proporciona los siguientes goals

  • build-jar

  • build-keystore

  • build-native: Genera un ejecutable en distintas plataformas, necesitará de software adicional para hacerlo, por ejemplo para generar un msi, precisa de wix

  • build-web: Genera un jnlp.

  • generate-key-store

  • help

  • jar: Genera un jar ejecutable en la ruta target/jfx/app/, ya que el jar normal no es ejecutable por carecer de MANIFEST valido.

  • list-bundlers: lista los posibles bundlers en la plataforma actual, por ejemplo para windowss, son: windows.app, msi, exe y jnlp.

  • native

  • run: Ejecuta la aplicacion

  • web

Note

Para poder ejecutar jnlp en local, hay que permitir la ejecucion del protocolo file:/ en la consola de configuracion de java

>"C:\Program Files\Java\jdk1.8.0_151\jre\bin\javacpl.exe"

4. Arquitectura

La clase principal de las aplicaciones JavaFX es javafx.application.Application, que representa la propia aplicación.

JavaFX presenta dos contenedores:

  • Stage: Representa la ventana principal, es recibido por parametro por el método start() del objeto Application.

  • Scene: Es el contenedor donde se incluyen los componentes UI. Son reemplazables por otros Scene. Su contenido se representa como un arbol de Node denominado Scene Graph.

Los Node dispone entre otros campos de: Id, style, effects, event handlers, state, etc.

4.1. Ciclo de Vida de Application

La clase javafx.application.Application dispone de los siguientes métodos para la gestion del ciclo de vida de la aplicación:

  • init(): Es llamado despues de crear la instancia de la clase Application. En este método no se puede acceder al API de JavaFX, no se pueden crear componnetes UI.

  • start(Stage stage): Es el lugar donde se crean los componentes UI, tambien se inicia el hilo Prism rendering thread que se encarga de la renderización. Recibe por parametro el objeto Stage.

public class Main extends Application {
    @Override
    public void start(Stage primaryStage) throws Exception {
    }
}

  • stop(): Es llamado antes de parar la aplicación. Es el lugar idoneo para liberar recursos captados por la aplicación.

  • static launch(): Arranca la aplicación JavaFX.

Primera aplicación
public class HelloWorld extends Application {

    @Override
    public void start(Stage primaryStage) {
        Button button = new Button("Hello World");
        button.setOnAction (e -> System.out.println("Hello World"));
        StackPane myPane = new StackPane();
        myPane.getChildren().add(button);
        Scene myScene = new Scene(myPane);
        primaryStage.setScene(myScene);
        primaryStage.setWidth(400);
        primaryStage.setHeight(300);
        primaryStage.show();
    }

    public static void main(String[] args) {
    launch(args);
    }
}

Se definen 5 tipos de Stage:

  • Decorated

  • Undecorated

  • Transparent

  • Unified

  • Utility

4.3. Scene

Es el contenedor donde se incluyen los componentes UI. Son reemplazables por otros Scene. Su contenido se representa como un arbol de Node denominado Scene Graph.

Para su definición se precisa establecer el Parent principal y el tamaño.

Creacion de un Scene empleando un objeto creado en el momento
StackPane root = new StackPane();
Scene scene = new Scene( root, 400, 200);

Los UIControl añadidos al Scene, pueden ser referenciados por una expresion CSS, por ejemplo para referenciar por id se emplea #.

Label label = (Label) scene.lookup("#label");

5. Interface de Usuario

Se definen las vista empleando ficheros FXML, que emplean un lenguaje de marcado propio de JavaFX

<?import javafx.geometry.Insets?>
<?import javafx.scene.layout.GridPane?>

<?import javafx.scene.control.Button?>
<?import javafx.scene.control.Label?>
<GridPane fx:controller="sample.Controller"
          xmlns:fx="http://javafx.com/fxml" alignment="center" hgap="10" vgap="10">
</GridPane>

Se puede asociar una clase Controladora a los componentes del FXML a través de la propiedad fx:controller, esta clase Controladora tiene como misión envolver todos los comportamientos asociados a la vista que representa el FXML.

Los ficheros FXML se pueden cargar empleando la clase FXMLLoader.

El método load(), carga el fichero FXML y retorna un objeto de tipo Parent.

public class Main extends Application {
    @Override
    public void start(Stage primaryStage) throws Exception {
        Parent root = FXMLLoader.load(getClass().getResource("sample.fxml"));
    }
}

La clase Parent pertenece a la jerarquia de Node y empieza la raiz de la jerarquia de elementos UI.

5.2.3. StackPane

Coloca los elementos en un pila, siendo el ultimo el visualizado.

private StackPane stackPane;

@Override
public void start(Stage primaryStage) throws Exception {
    VBox root = new VBox();

    // StackPane
    stackPane = new StackPane();
    stackPane.setPrefSize(300, 150);
    stackPane.setStyle("-fx-background-color: Gainsboro;-fx-border-color: blue;");

    // Add Label to StackPane
    Label label = new Label("I'm a Label");
    label.setStyle("-fx-background-color:yellow");
    label.setPadding(new Insets(5,5,5,5));
    stackPane.getChildren().add(label);

    //Los componentes solamente estan tapados, pero siguen visibles, para que se oculten
    label.setVisible(false);

    // Add Button to StackPane
    Button button = new Button("I'm a Button");
    button.setStyle("-fx-background-color: cyan");
    button.setPadding(new Insets(5,5,5,5));
    stackPane.getChildren().add(button);

    button.setVisible(false);

    // Add CheckBox to StackPane
    CheckBox checkBox = new CheckBox("I'm a CheckBox");
    checkBox.setOpacity(1);
    checkBox.setStyle("-fx-background-color:olive");
    checkBox.setPadding(new Insets(5,5,5,5));
    stackPane.getChildren().add(checkBox);

    checkBox.setVisible(true);

    // Add StackPane to VBox
    root.getChildren().add(stackPane);

    // Add Button Change to VBox
    Button controlButton = new Button("Change Top");
    controlButton.setOnAction(new EventHandler<ActionEvent>() {
        @Override
        public void handle(ActionEvent event) {
            changeTop();
        }
    });
    root.getChildren().add(controlButton);

    //Aligment components in VBox
    root.setAlignment(Pos.CENTER);
    VBox.setMargin(stackPane, new Insets(10, 10, 10, 10));
    VBox.setMargin(controlButton, new Insets(10, 10, 10, 10));

    Scene scene = new Scene(root, 550, 250);
    primaryStage.setTitle("StackPane");
    primaryStage.setScene(scene);
    primaryStage.show();
}

private void changeTop() {
    ObservableList<Node> childs = this.stackPane.getChildren();
    if (childs.size() > 1) {
        Node topNode = childs.get(childs.size()-1);
        topNode.setVisible(false);

        // This node will be brought to the front
        Node newTopNode = childs.get(childs.size()-2);
        newTopNode.setVisible(true);

        topNode.toBack();
    }
}

5.2.8. AnchorPane

Permite anclar los componentes a los cuatro lados del Layout: top, right, bottom y left. Ante un redimensionamiento del panel, los componentes mantienen la distancia al panel segun las anclas definidas.

AnchorPane root = new AnchorPane();
Button button1 = new Button("(B1) Top + Left + Right");
Button button2 = new Button("(B2) Top + Left + Right");
Button button3 = new Button("(B3) Top + Left + Right");
Button button4 = new Button("(B4) Top + Left + Right + Bottom");

// (B1) Anchor to the Top + Left + Right
AnchorPane.setTopAnchor(button1, 40.0);
AnchorPane.setLeftAnchor(button1, 50.0);
AnchorPane.setRightAnchor(button1, 70.0);

// (B2) Anchor to the Top + Left + Right
AnchorPane.setTopAnchor(button2, 90.0);
AnchorPane.setLeftAnchor(button2, 10.0);
AnchorPane.setRightAnchor(button2, 320.0);

// (B3) Anchor to the Top + Left + Right
AnchorPane.setTopAnchor(button3, 100.0);
AnchorPane.setLeftAnchor(button3, 250.0);
AnchorPane.setRightAnchor(button3, 20.0);

// (B4) Anchor to the four sides of AnchorPane
AnchorPane.setTopAnchor(button4, 150.0);
AnchorPane.setLeftAnchor(button4, 40.0);
AnchorPane.setRightAnchor(button4, 50.0);
AnchorPane.setBottomAnchor(button4, 45.0);

// Add buttons to AnchorPane
root.getChildren().addAll(button1, button3, button2, button4);

Scene scene = new Scene(root, 550, 250);
primaryStage.setTitle("AnchorPane Layout Demo (o7planning.org)");
primaryStage.setScene(scene);
primaryStage.show();

5.2.10. ScrollPane

Permite añadir barras de scroll alrededor de un componente UI

Image roses = new Image(getClass().getResourceAsStream("roses.jpg"));
ScrollPane sp = new ScrollPane();
sp.setContent(new ImageView(roses));

si se establece la propiedad Pannable, se permite mover los scroll arrastrando con el raton

sp.setPannable(true);

Se puede definir cuando apareceran los scroll

sp.setHbarPolicy(ScrollPane.ScrollBarPolicy.NEVER);
sp.setVbarPolicy(ScrollPane.ScrollBarPolicy.ALWAYS);

Se puede hacer que el tamaño del ScrollPane se adapte al contenido, sin mostrar las barras con setFitToWidth y setFitToHeight

5.3. Controles UI

La clase base es Control.

5.3.1. ImageView

Componente que permite representar Image.

Un Image se puede crear a partir de la url o de un InputStream, indicando aspectos como:

  • Tamaño

  • Preservar aspecto, para que no se deforme la imagen al representarla.

  • Aplicar algoritmos que mejoren el escalado.

Para este ejemplo debe existir en el classpath en el paquete sample la imagen java-32.png
InputStream input= getClass().getResourceAsStream("/sample/java-32.png");
Image image = new Image(input);
ImageView imageView = new ImageView(image);

InputStream input2 = clazz.getResourceAsStream("/sample/java-32.png");
Image image2 = new Image(input2, 300, 200, false, true);
ImageView imageView2 = new ImageView(image2);

5.3.2. Label

Componente que permite representar un texto no editable, un icono o ambos.

// Image Source
InputStream input= getClass().getResourceAsStream("/sample/java-32.png");
Image image = new Image(input);
ImageView imageView = new ImageView(image);
Label label = new Label("JavaFX");
// Set Image
label.setGraphic(imageView);

Se pueden definir tipos de letras distintos con la propiedad Font

label.setFont(new Font("Arial", 30));

label.setFont(Font.font("Cambria", 32));

Se puede definir el color de relleno con la propiedad TextFill

label.setTextFill(Color.web("#0076a3"));

Se puede hacer que el texto de la etiqueta se represente en varias lineas si es que es muy largo en comparación al espacio destinado a la Label

label.setWrapText(true);

5.3.3. Hyperlink

Componente que permite mostrar un enlace a un recurso localo remoto.

Tiene propiedades como:

  • visited: Indica si ha sido pulsado, cambiando el color.

  • underline: Muestra el enlace subrayado, para resaltar que es un enlace.

Hyperlink hyperlink = new Hyperlink("pulsa");
hyperlink.setOnAction(new EventHandler<ActionEvent>() {
    @Override
    public void handle(ActionEvent event) {
        getHostServices().showDocument("https://eclipse.org");
    }
});
Button button1 = new Button("On/Off Visited");
button1.setOnAction(new EventHandler<ActionEvent>() {
    @Override
    public void handle(ActionEvent event) {
        boolean visited = hyperlink.isVisited();
        hyperlink.setVisited(!visited);
        hyperlink.setText("pulsa "+" (visited:"+ hyperlink.isVisited()+")");
    }
});
Button button2 = new Button("On/Off Underline");
button2.setOnAction(new EventHandler<ActionEvent>() {
    @Override
    public void handle(ActionEvent event) {
        boolean underline = hyperlink.isUnderline();
        hyperlink.setUnderline(!underline);
        hyperlink.setText("pulsa "+" (underline:"+ hyperlink.isUnderline()+")");
    }
});

5.3.6. ToggleButton

Boton con dos estados selected y deselected.

Pueden ser agrupados e ToggleGroup de tal forma que solo uno de los botones agrupados puede estar selected.

CellFactory

Permite cambiar el aspecto de los item del ComboBox.

Se basa en la interface Callback, interface parametrica con un unico método call().

Se debe crear una implementacion de dicha interface e implementar call() para que retorne un objeto ListCell.

La clase ListCell dispone de un método updateItem() que es invocado en el pintado del ComboBox y que será el lugar donde implementar el cambio de estilos de los item.

ComboBox<String> myComboBox = new ComboBox<String>();

myComboBox.setCellFactory(
    new Callback<ListView<String>, ListCell<String>>() {
        @Override
        public ListCell<String> call(ListView<String> param) {
            final ListCell<String> cell = new ListCell<String>() {
                {
                    super.setPrefWidth(100);
                }

                @Override
                public void updateItem(String item, boolean empty) {
                    super.updateItem(item, empty);
                    if (item != null) {
                        setText(item);
                        if (item.contains("A")) {
                            setTextFill(Color.RED);
                        } else if (item.contains("B")) {
                            setTextFill(Color.GREEN);
                        } else {
                            setTextFill(Color.BLACK);
                        }
                    } else {
                        setText(null);
                    }
                }
            };
            return cell;
        }
    }
);

Callback<DatePicker, DateCell> dayCellFactory =

                        || item.getDayOfWeek() == DayOfWeek.TUESDAY
                        || item.getDayOfWeek() == DayOfWeek.WEDNESDAY) {
                    setDisable(true);
                    setStyle("-fx-background-color: #ffc0cb;");

        };

};

}

5.6.2. Binding (Enlaces)

El API de Binding, permite enlazar propiedades de unos objetos con propiedades de otros.

El API se divide en dos:

  • Fluent API.

  • Clase Bindings: Clase con métodos estaticos que permiten construir expresiones de enlazado.

FLuent API

Metodos que presentan las clases Property para realizar enlazados. Les hay de dos tipos:

  • Enlazados directos de propiedades, que permiten cambiar el valor de una propiedad de un objeto cuando cambie otra propiedad de ese mismo objeto o de otro.

  • bind()

  • unbind()

  • binBidirectional()

  • unbindBidirectional()

  • Cambios de propiedades empleando operaciones predefinidas, algunos ejemplos son:

  • IntegerProperty.add(),

  • IntegerProperty.divide(),

  • IntegerProperty.multiply(),

  • StringProperty.isNull(),

  • StringProperty.isEmpty(),

  • StringProperty.isEqualTo(), etc.

Estos métodos, generan un objeto de que sigue la jerarquia de la interface Binding:

  • StringBinding

  • BooleanBinding

  • ListBinding

  • ObjectBinding

  • SetBinding

  • MapBinding

  • NumberBinding

    • IntegerBinding

    • DoubleBinding

    • FloatBinding

    • LongBinding

Los objetos Binding mantienen el enlace definido mientras existan.

//Definicion de propiedades que participan del enlazado
IntegerProperty num1 = new SimpleIntegerProperty(1);
IntegerProperty num2 = new SimpleIntegerProperty(2);
IntegerProperty num3 = new SimpleIntegerProperty(3);

//Definicion del enlazado
NumberBinding sum = num1.add(num2).add(num3);

//Modificacion de propiedades, comprobando que el enlazado cambia.
System.out.println(sum.getValue()); //6
num1.set(2);
System.out.println(sum.getValue());//7
num3.set(6);
System.out.println(sum.getValue());//10
Clase Bindings

Permite realizar las mismas operaciones que al Fluent API, pero de forma independiente.

//Definicion de propiedades que participan del enlazado
IntegerProperty num1 = new SimpleIntegerProperty(1);
IntegerProperty num2 = new SimpleIntegerProperty(2);

//Definicion del enlazado
NumberBinding sum = Bindings.add(num1,num2);

//Modificacion de propiedades, comprobando que el enlazado cambia.
System.out.println(sum.getValue());
num1.setValue(2);
System.err.println(sum.getValue());

E incluso se pueden mezclar

//Definicion de propiedades que participan del enlazado
IntegerProperty num1 = new SimpleIntegerProperty(1);
IntegerProperty num2 = new SimpleIntegerProperty(2);
IntegerProperty num3 = new SimpleIntegerProperty(3);
IntegerProperty num4 = new SimpleIntegerProperty(4);

//Definicion del enlazado
NumberBinding total = Bindings.add(num1.multiply(num2),num3.multiply(num4));

//Modificacion de propiedades, comprobando que el enlazado cambia.
System.out.println(total.getValue());
num1.setValue(2);
System.err.println(total.getValue());

Se pueden definir Listener sobre los Binding de tipo:

  • InvalidationListener (calculo perezoso): Cuando se define y se cambia algun valor de los elementos que participan del Binding, se ejecuta este Listener, el calculo del Binding no se produce, perezoso, y se cambia la propiedad valid del Observable a false. Si el Binding es invalido, no se vuelve a lanzar el evento. Cuando se calcula el valor, la propiedad valid se pone a true, ya que se realiza el calculo que antes se quedo sin hacer.

  • ChangeListener (calculo inmediato). Permiten escuchar cambios en los elementos del Binding. Realiza los calculos de forma inmediata no empleandose la propiedad valid

Jerarquia de Clases Binding

Si se desea mas control sobre los Binding se puede extender alguna clase de la jerarquia:

  • StringBinding

  • BooleanBinding

  • ListBinding

  • ObjectBinding

  • SetBinding

  • MapBinding

  • NumberBinding

    • IntegerBinding

    • DoubleBinding

    • FloatBinding

    • LongBinding

Implementando el método computeValue(), que es quien realiza el calculo de enlazado.

DoubleProperty a = new SimpleDoubleProperty(1);
DoubleProperty b = new SimpleDoubleProperty(2);
DoubleProperty c = new SimpleDoubleProperty(3);
DoubleProperty d = new SimpleDoubleProperty(4);

DoubleBinding db = new DoubleBinding() {
    {
        super.bind(a, b, c, d);
    }

    @Override
    protected double computeValue() {
        return (a.get() * b.get()) + (c.get() * d.get());
    }
};

System.out.println(db.get());
b.set(3);
System.out.println(db.get());

  • RadioMenuItem: Permite elegir entre varias opciones cual es la marcada.

RadioMenuItem updateItem1 = new RadioMenuItem("Auto Update");
RadioMenuItem updateItem2 = new RadioMenuItem("Ask for Update");
ToggleGroup group = new ToggleGroup();

5.9.1. Shortcuts

Se pueden definir combinaciones de teclas para acceder mas rapido a los menus. Para ello unicamente hay que definir la expresion de la combinacion de teclas con un objeto KeyCombination y asociarla con setAccelerator()

exitItem.setAccelerator(KeyCombination.keyCombination("Ctrl+X"));
exitItem.setOnAction(new EventHandler<ActionEvent>() {
    @Override
    public void handle(ActionEvent event) {
        System.exit(0);
    }
});

5.10. ContextMenu

Un menu constextual es aquel que se asocia a un componente visual y no a una ventana.

No se muestra por defecto, solo cuando se interaciona con el componente al que está asociado pulsado con el boton derecho del raton.

Se pueden asociar varios MenuItem, de los tipo: MenuItem, CheckMenuItem, RadioMenuItem, SeparatorMenuItem y ContextMenu.

MenuItem item1 = new MenuItem("Menu Item 1");
MenuItem item2 = new MenuItem("Menu Item 2");
contextMenu.getItems().addAll(item1, item2);

image.setOnContextMenuRequested(new EventHandler<ContextMenuEvent>() {
    @Override
    public void handle(ContextMenuEvent event) {
        contextMenu.show(image, event.getScreenX(), event.getScreenY());
    }
});

Al igual que con el MenuBar se pueden añadir Accelerator a los MenuItem.

Se pueden escuchar los siguientes eventos sobre el ContextMenu

  • OnShowing: Se invoca justo antes de ser mostrado.

  • OnShown: Se invoca justo despues de ser mostrado.

  • OnHiding: Se invoca justo antes de ser ocultado.

  • OnHidden: Se invoca justo despues de ser ocultado.

  • OnCloseRequest: Se invoca cuando un evento externo hace que se cierre.

6. Eventos

Los componentes Node generan eventos, que se representan con las clase javafx.event.Event.

Algunos de los eventos mas habituales son: ActionEvent,TreeModificationEvent, InputEvent, ListView.EditEvent, MediaErrorEvent, TableColumn.CellEditEvent, TreeItem.TreeModificationEvent, TreeView.EditEvent, WebEvent, WindowEvent, WorkerStateEvent, etc.

El proceso de entrega del envento, se divide en cuatro fases:

  • Selección del Nodo Destino (Target): Se determina que elemento de la UI es sobre el que se ha producido el evento.

  • Construcción de la ruta: Es el camino dentro del arbol de componentes que se ha de seguir entre el componente que produce el evento y la raiz del arbol. La ruta se puede modificar a medida que los Filters y los Handdlers a lo largo de la ruta procesan el evento. Además, si un Filter o Handdler consume el evento en cualquier momento, es posible que algunos nodos de la ruta inicial no reciban el evento, porque interrumpa la propagacion.

  • Captura de eventos: El nodo raíz de su aplicación distribuye el evento y pasa la cadena de distribución de eventos al nodo de destino (Target). En el camino cualquier componente puede, a traves de un Filter capturar el evento.

  • Evento burbujeante (propagacion): El camino inverso, esta vez desde el destino (Target) al raiz. En este caso son los Handler los que pueden capturar el evento para procesarlo.

Cuando un Filter o Handler trata un evento, puede llamar al método consume(), que finaliza el viaje del evento por la ruta.

new EventHandler<KeyEvent>() {
    public void handle(final KeyEvent keyEvent) {
        keyEvent.consume();
    }
}

6.1. Filtros

Un nodo puede registrar más de un Filter. El orden en que se los llama se basa en la jerarquía de los tipos de eventos. Los Filter para un tipo de evento específico se ejecutan antes que los Filter para los tipos de evento genéricos. Por ejemplo, MouseEvent.MOUSE_PRESSED se llama a un filtro para el evento antes del filtro para el InputEvent.ANY. El orden en el que se ejecutan dos filtros en el mismo nivel no se especifica.

6.2. Manejadores (EventHandler)

Un nodo puede registrar más de un Handler. El orden en que se los llama se basa en la jerarquía de los tipos de eventos. Los Handler para un tipo de evento específico se ejecutan antes que los Handler para los tipos de eventos genéricos. Por ejemplo, un controlador para el KeyEvent.KEY_TYPED se llama antes que el controlador para el InputEvent.ANY. El orden en el que se ejecutan dos manejadores en el mismo nivel no se especifica.

Se proporcioan dos formas de registrar Handler,

  • La generica a traves del método addEventHandler(Event, EventHandler).

Button btn = new Button();
btn.setLayoutX(100);
btn.setLayoutY(80);
btn.setText("Hello World");
btn.addEventHandler(ActionEvent.ACTION, new EventHandler<ActionEvent>() {
    public void handle(ActionEvent event) {
        System.out.println("Hello World");
    }
});

  • Con los métodos setOn<Event>(EventHandler).

Button btn = new Button();
btn.setLayoutX(100);
btn.setLayoutY(80);
btn.setText("Hello World");
btn.setOnAction(new EventHandler<ActionEvent>() {
    public void handle(ActionEvent event) {
        System.out.println("Hello World");
    }
});

Para los EventHandler se pueden emplear expresiones lambda a partir de java 8.

Button button = new Button("Hello World");
button.setOnAction (e -> System.out.println("Hello World"));

Se pueden eliminar Handler

btn.removeEventHandler(ActionEvent.ACTION, handler);

6.3. Controladores

Permiten definir comportamientos dinamicos sobre los controles UI definidos en un FXML.

Para tener la referencia a los componentes desde el controlador, se emplea la anotacion @FXML con atributos de la clase controlador del tipo los componentes UI.

public class Controller {
    @FXML //fx:id="gridPane"
    private GridPane gridPane;
}

Donde para el ejemplo anterior se tiene el siguiente FXML

<GridPane fx:controller="sample.Controller"
          xmlns:fx="http://javafx.com/fxml"
          alignment="center"
          hgap="10"
          vgap="10"
          fx:id="gridPane">
</GridPane>

De la misma forma, con la anoacion @FXML se pueden asignar métodos del controlador, como controladores de eventos EventHandler

<Button fx:id="boton" onAction="#doAction" text="Pulsar" />
@FXML
void doAction (ActionEvent event) {}

Si se precisa obtener la referencia al controlador desde Application se puede a través de FXMLLoader

URL location = getClass().getResource("sample.fxml");
FXMLLoader fxmlLoader = new FXMLLoader(location);
Parent root = fxmlLoader.load();
Controller controller = (Controller)fxmlLoader.getController();

De la misma forma si se desea establecer el controlador de forma progrmatica desde Application se hace desde el mismo objeto

fxmlLoader.setController(this);

Si se necesita inicializar algun componente, se puede implementar la interface Initializable.

public class InitController implements Initializable{
        @Override
        public void initialize(URL location, ResourceBundle resources) {
                Planet mercury = new Planet("MER", "Mercury");
                Planet venus = new Planet("VEN", "Venus");
                Planet earth = new Planet("EAR", "Earth");

                ObservableList<Planet> list = FXCollections.observableArrayList(mercury, venus, earth);
                comboBox.setItems(list);
        }
}

7. Estilos CSS

Los componentes Node pueden definir su representación visual empleando CSS.

Para asignar un CSS a un Scene se emplea

scene.getStylesheets().add(getClass().getResource("application.css").toExternalForm());

La definicion de estilos es como en los CSS aplicados a los HTML, la diferencia aparece en los identificadores de las propiedades, que son precedidas con -fx

#label {
        -fx-text-fill: red;
}

Se pueden asignar estilos de forma programatica a los componentes con el método setStyle()

label.setStyle("-fx-text-fill: white;");

button1.setStyle("-fx-font: 22 arial; -fx-base: #b6e7c9;");

19.4. Ejercicio SVG

Se desea una aplicacion que represente una camiseta con un logo, pudiendose cambiar el logo y el color de la camiseta con un combo.

public class ColorPickerSample extends Application {

    ImageView logo = new ImageView(
        new Image(getClass().getResourceAsStream("OracleLogo.png"))
    );

    public static void main(String[] args) {
        launch(args);
    }

    @Override
    public void start(Stage stage) {
        stage.setTitle("ColorPicker");

        Scene scene = new Scene(new VBox(20), 300, 300);
        scene.setFill(Color.web("#ccffcc"));
        VBox box = (VBox) scene.getRoot();

        ToolBar tb = new ToolBar();
        box.getChildren().add(tb);

        final ComboBox logoSamples = new ComboBox();
        logoSamples.getItems().addAll(
                "Oracle",
                "Java",
                "JavaFX",
                "Cup");
        logoSamples.setValue("Oracle");

        logoSamples.valueProperty().addListener(new ChangeListener<String>() {
            @Override
            public void changed(ObservableValue ov, String t, String t1) {
                logo.setImage(
                    new Image(getClass().getResourceAsStream(t1+"Logo.png"))
                );
            }
        });


        final ColorPicker colorPicker = new ColorPicker();
        tb.getItems().addAll(logoSamples, colorPicker);

        StackPane stack = new StackPane();
        box.getChildren().add(stack);

        final SVGPath svg = new SVGPath();
        svg.setContent("M70,50 L90,50 L120,90 L150,50 L170,50"
            + "L210,90 L180,120 L170,110 L170,200 L70,200 L70,110 L60,120 L30,90"
            + "L70,50");
        svg.setStroke(Color.DARKGREY);
        svg.setStrokeWidth(2);
        svg.setEffect(new DropShadow());
        svg.setFill(colorPicker.getValue());
        stack.getChildren().addAll(svg, logo);

        colorPicker.setOnAction(new EventHandler() {
            public void handle(Event t) {
                svg.setFill(colorPicker.getValue());
            }
        });

        stage.setScene(scene);
        stage.show();
    }
}
