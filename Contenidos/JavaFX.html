1. Introduccion

Framework para el desarrollo de aplicaciones de escritorio, embebidas y de movilidad.

Forma parte de la JDK 8.

Ofrece APIs como:

  • Graficos 2D y 3D.

  • UI controls, layouts y charts

  • Soporte para Audio y video

  • Efectos y animaciones

  • Soporte para HTML5 y CSS

Proporciona alto rendimiento al emplear motores nativos de renderizado, como Direct3D para Windows y OpenGL para Linux.

Tiene compatibilidad con APIs como Swing.

Emplea un lenguaje de marcado FXML para crear las vistas, equivalente a los HMTL del mundo web.

2. Instalacion

Para trabajar con JavaFX se necesita

  • JDK 8, que incluye el SDK de JavaFX, que se puede descargar de aquí

  • IDE con soporte para JavaFX: Eclipse, Netbeans, Intellij, etc

  • JavaFX Scene Builder: es una herramienta proporcionada por Oracle, que actualmente está en la version 2.0, para el diseño de ventanas JavaFX. Se puede descargar de aquí.

  • Se dispone de una distribucion de eclipse, con los plugin necesarios instalados aquí == Arquitectura

La clase principal de las aplicaciones JavaFX es javafx.application.Application, que representa la propia aplicación.

JavaFX presenta dos contenedores:

  • Stage: Representa la ventana principal, es recibido por parametro por el método start() del objeto Application.

  • Scene: Es el contenedor donde se incluyen los componentes UI. Son reemplazables por otros Scene. Su contenido se representa como un arbol de Node denominado Scene Graph.

Los Node dispone entre otros campos de: Id, style, effects, event handlers, state, etc.

2.1. Ciclo de Vida de Application

La clase javafx.application.Application dispone de los siguientes métodos para la gestion del ciclo de vida de la aplicación:

  • init(): Es llamado despues de crear la instancia de la clase Application. En este método no se puede acceder al API de JavaFX, no se pueden crear componnetes UI.

  • start(Stage stage): Es el lugar donde se crean los componentes UI, tambien se inicia el hilo Prism rendering thread que se encarga de la renderización. Recibe por parametro el objeto Stage.

public class Main extends Application {
    @Override
    public void start(Stage primaryStage) throws Exception {
    }
}

  • stop(): Es llamado antes de parar la aplicación. Es el lugar idoneo para liberar recursos captados por la aplicación.

  • static launch(): Arranca la aplicación JavaFX.

Primera aplicación
public class HelloWorld extends Application {

    @Override
    public void start(Stage primaryStage) {
        Button button = new Button("Hello World");
        button.setOnAction (e -> System.out.println("Hello World"));
        StackPane myPane = new StackPane();
        myPane.getChildren().add(button);
        Scene myScene = new Scene(myPane);
        primaryStage.setScene(myScene);
        primaryStage.setWidth(400);
        primaryStage.setHeight(300);
        primaryStage.show();
    }

    public static void main(String[] args) {
    launch(args);
    }
}

3. Interface de Usuario

Se definen las vista empleando ficheros FXML, que emplean un lenguaje de marcado propio de JavaFX

<?import javafx.geometry.Insets?>
<?import javafx.scene.layout.GridPane?>

<?import javafx.scene.control.Button?>
<?import javafx.scene.control.Label?>
<GridPane fx:controller="sample.Controller"
          xmlns:fx="http://javafx.com/fxml" alignment="center" hgap="10" vgap="10">
</GridPane>

Se puede asociar una clase Controladora a los componentes del FXML a través de la propiedad fx:controller, esta clase Controladora tiene como misión envolver todos los comportamientos asociados a la vista que representa el FXML.

Los ficheros FXML se pueden cargar empleando la clase FXMLLoader.

El método load(), carga el fichero FXML y retorna un objeto de tipo Parent.

public class Main extends Application {
    @Override
    public void start(Stage primaryStage) throws Exception {
        Parent root = FXMLLoader.load(getClass().getResource("sample.fxml"));
    }
}

3.2.3. StackPane

Coloca los elementos en un pila, siendo el ultimo el visualizado.

private StackPane stackPane;

@Override
public void start(Stage primaryStage) throws Exception {
    VBox root = new VBox();

    // StackPane
    stackPane = new StackPane();
    stackPane.setPrefSize(300, 150);
    stackPane.setStyle("-fx-background-color: Gainsboro;-fx-border-color: blue;");

    // Add Label to StackPane
    Label label = new Label("I'm a Label");
    label.setStyle("-fx-background-color:yellow");
    label.setPadding(new Insets(5,5,5,5));
    stackPane.getChildren().add(label);

    //Los componentes solamente estan tapados, pero siguen visibles, para que se oculten
    label.setVisible(false);

    // Add Button to StackPane
    Button button = new Button("I'm a Button");
    button.setStyle("-fx-background-color: cyan");
    button.setPadding(new Insets(5,5,5,5));
    stackPane.getChildren().add(button);

    button.setVisible(false);

    // Add CheckBox to StackPane
    CheckBox checkBox = new CheckBox("I'm a CheckBox");
    checkBox.setOpacity(1);
    checkBox.setStyle("-fx-background-color:olive");
    checkBox.setPadding(new Insets(5,5,5,5));
    stackPane.getChildren().add(checkBox);

    checkBox.setVisible(true);

    // Add StackPane to VBox
    root.getChildren().add(stackPane);

    // Add Button Change to VBox
    Button controlButton = new Button("Change Top");
    controlButton.setOnAction(new EventHandler<ActionEvent>() {
        @Override
        public void handle(ActionEvent event) {
            changeTop();
        }
    });
    root.getChildren().add(controlButton);

    //Aligment components in VBox
    root.setAlignment(Pos.CENTER);
    VBox.setMargin(stackPane, new Insets(10, 10, 10, 10));
    VBox.setMargin(controlButton, new Insets(10, 10, 10, 10));

    Scene scene = new Scene(root, 550, 250);
    primaryStage.setTitle("StackPane");
    primaryStage.setScene(scene);
    primaryStage.show();
}

private void changeTop() {
    ObservableList<Node> childs = this.stackPane.getChildren();
    if (childs.size() > 1) {
        Node topNode = childs.get(childs.size()-1);
        topNode.setVisible(false);

        // This node will be brought to the front
        Node newTopNode = childs.get(childs.size()-2);
        newTopNode.setVisible(true);

        topNode.toBack();
    }
}

});

3.6.2. Binding (Enlaces)

El API de Binding, permite enlazar propiedades de unos objetos con propiedades de otros.

El API se divide en dos:

  • Fluent API.

  • Clase Bindings: Clase con métodos estaticos que permiten construir expresiones de enlazado.

FLuent API

Metodos que presentan las clases Property para realizar enlazados. Les hay de dos tipos:

  • Enlazados directos de propiedades, que permiten cambiar el valor de una propiedad de un objeto cuando cambie otra propiedad de ese mismo objeto o de otro.

  • bind()

  • unbind()

  • binBidirectional()

  • unbindBidirectional()

  • Cambios de propiedades empleando operaciones predefinidas, algunos ejemplos son:

  • IntegerProperty.add(),

  • IntegerProperty.divide(),

  • IntegerProperty.multiply(),

  • StringProperty.isNull(),

  • StringProperty.isEmpty(),

  • StringProperty.isEqualTo(), etc.

Estos métodos, generan un objeto de que sigue la jerarquia de la interface Binding:

  • StringBinding

  • BooleanBinding

  • ListBinding

  • ObjectBinding

  • SetBinding

  • MapBinding

  • NumberBinding

    • IntegerBinding

    • DoubleBinding

    • FloatBinding

    • LongBinding

Los objetos Binding mantienen el enlace definido mientras existan.

//Definicion de propiedades que participan del enlazado

//Definicion del enlazado
NumberBinding sum = num1.add(num2).add(num3);

Permite realizar las mismas operaciones que al Fluent API, pero de forma independiente.

Se pueden definir Listener sobre los Binding de tipo:

  • InvalidationListener (calculo perezoso): Cuando se define y se cambia algun valor de los elementos que participan del Binding, se ejecuta este Listener, el calculo del Binding no se produce, perezoso, y se cambia la propiedad valid del Observable a false. Si el Binding es invalido, no se vuelve a lanzar el evento. Cuando se calcula el valor, la propiedad valid se pone a true, ya que se realiza el calculo que antes se quedo sin hacer.

  • ChangeListener (calculo inmediato). Permiten escuchar cambios en los elementos del Binding. Realiza los calculos de forma inmediata no empleandose la propiedad valid

4. Eventos

Los componentes Node generan eventos, que se representan con las clase javafx.event.Event.

Algunos de los eventos mas habituales son: ActionEvent,TreeModificationEvent, InputEvent, ListView.EditEvent, MediaErrorEvent, TableColumn.CellEditEvent, TreeItem.TreeModificationEvent, TreeView.EditEvent, WebEvent, WindowEvent, WorkerStateEvent, etc.

El proceso de entrega del envento, se divide en cuatro fases:

  • Selección del Nodo Destino (Target): Se determina que elemento de la UI es sobre el que se ha producido el evento.

  • Construcción de la ruta: Es el camino dentro del arbol de componentes que se ha de seguir entre el componente que produce el evento y la raiz del arbol. La ruta se puede modificar a medida que los Filters y los Handdlers a lo largo de la ruta procesan el evento. Además, si un Filter o Handdler consume el evento en cualquier momento, es posible que algunos nodos de la ruta inicial no reciban el evento, porque interrumpa la propagacion.

  • Captura de eventos: El nodo raíz de su aplicación distribuye el evento y pasa la cadena de distribución de eventos al nodo de destino (Target). En el camino cualquier componente puede, a traves de un Filter capturar el evento.

  • Evento burbujeante (propagacion): El camino inverso, esta vez desde el destino (Target) al raiz. En este caso son los Handler los que pueden capturar el evento para procesarlo.

Cuando un Filter o Handler trata un evento, puede llamar al método consume(), que finaliza el viaje del evento por la ruta.

new EventHandler<KeyEvent>() {
    public void handle(final KeyEvent keyEvent) {
        keyEvent.consume();
    }
}

4.1. Filtros

Un nodo puede registrar más de un Filter. El orden en que se los llama se basa en la jerarquía de los tipos de eventos. Los Filter para un tipo de evento específico se ejecutan antes que los Filter para los tipos de evento genéricos. Por ejemplo, MouseEvent.MOUSE_PRESSED se llama a un filtro para el evento antes del filtro para el InputEvent.ANY. El orden en el que se ejecutan dos filtros en el mismo nivel no se especifica.

4.2. Manejadores (EventHandler)

Un nodo puede registrar más de un Handler. El orden en que se los llama se basa en la jerarquía de los tipos de eventos. Los Handler para un tipo de evento específico se ejecutan antes que los Handler para los tipos de eventos genéricos. Por ejemplo, un controlador para el KeyEvent.KEY_TYPED se llama antes que el controlador para el InputEvent.ANY. El orden en el que se ejecutan dos manejadores en el mismo nivel no se especifica.

Se proporcioan dos formas de registrar Handler,

  • La generica a traves del método addEventHandler(Event, EventHandler).

Button btn = new Button();
btn.setLayoutX(100);
btn.setLayoutY(80);
btn.setText("Hello World");
btn.addEventHandler(ActionEvent.ACTION, new EventHandler<ActionEvent>() {
    public void handle(ActionEvent event) {
        System.out.println("Hello World");
    }
});

  • Con los métodos setOn<Event>(EventHandler).

Button btn = new Button();
btn.setLayoutX(100);
btn.setLayoutY(80);
btn.setText("Hello World");
btn.setOnAction(new EventHandler<ActionEvent>() {
    public void handle(ActionEvent event) {
        System.out.println("Hello World");
    }
});

Para los EventHandler se pueden emplear expresiones lambda a partir de java 8.

Button button = new Button("Hello World");
button.setOnAction (e -> System.out.println("Hello World"));

Se pueden eliminar Handler

btn.removeEventHandler(ActionEvent.ACTION, handler);

  • Task

Se definen distintos hilos para arrancar y renderizar la aplicación.

9.1. Worker

Define un API para poder comunicar al hilo en segundo plano con la UI.

Define una propiedad State que es observable, la cual describe el ciclo de vida del Worker

  • READY: Preparado para ejecutarse

  • SCHEDULED: Planificada su ejecucion

  • RUNNING: En ejecucion

  • SUCCEEDED: Finalizado de forma existosa

  • FAILED: Finalizado con errores

  • CANCELLED: Cancelado

Ejemplo de monitorizacion de un Worker, en este caso, el que internamente define WebEngine para cargar las nuevas paginas HTML
public class WebViewWithProgressDemo extends Application {
    @Override
    public void start(final Stage stage) {
        TextField addressBar = new TextField();
        addressBar.setText("https://eclipse.org");

        Button goButton = new Button("Go!");

        Label stateLabel = new Label();
        stateLabel.setTextFill(Color.RED);

        ProgressBar progressBar = new ProgressBar();

        final WebView browser = new WebView();
        final WebEngine webEngine = browser.getEngine();

        // A Worker load the page
        Worker<Void> worker = webEngine.getLoadWorker();

        // Listening to the status of worker
        worker.stateProperty().addListener(
            new ChangeListener<State>() {
                @Override
                public void changed(ObservableValue<? extends State> observable, State oldValue, State newValue) {
                    stateLabel.setText("Loading state: " + newValue.toString());
                    if (newValue == Worker.State.SUCCEEDED) {
                        stage.setTitle(webEngine.getLocation());
                        stateLabel.setText("Finish!");
                    }
                }
            }
        );

        // Bind the progress property of ProgressBar with progress property of Worker
        progressBar.progressProperty().bind(worker.progressProperty());

        goButton.setOnAction(
            new EventHandler<ActionEvent>() {
                @Override
                public void handle(ActionEvent event) {
                    String url = addressBar.getText();
                    // Load the page.
                    webEngine.load(url);
                }
            }
        );

        VBox root = new VBox();
        root.getChildren().addAll(addressBar, goButton, stateLabel, progressBar, browser);
        Scene scene = new Scene(root);
        stage.setTitle("JavaFX WebView (o7planning.org)");
        stage.setScene(scene);
        stage.setWidth(450);
        stage.setHeight(300);
        stage.show();
    }

    public static void main(String[] args) {
        launch(args);
    }
}

9.2. Task

Implementancion de java.util.concurrent.FutureTask que permite definir las tareas de larga duracion.

La tarea de larga duración se definirá en el mátodo call() de una nueva implementacion de la clase Task.

Task<Integer> task = new Task<Integer>() {
    @Override protected Integer call() throws Exception {
        //Aqui iria la tarea de larga duracion
        return 0;
    }
};

En el método call() no se podrá acceder a componnete de UI, para poder actualizar la UI se debera invocar a los métodos updateProgress, updateMessage o updateTitle, que actualizan unas propiedades del objeto Task que pueden ser bindeados hacia componentes UI.

Task task = new Task<Void>() {
    @Override public Void call() {
        static final int max = 1000000;
        for (int i=1; i<=max; i++) {
            if (isCancelled()) {
               break;
            }
            updateProgress(i, max);
        }
        return null;
    }
};
ProgressBar bar = new ProgressBar();
bar.progressProperty().bind(task.progressProperty());
new Thread(task).start();

Los Task no son reutilizables.

Para ejecutar el Task, se puede optar por dos formas

  • Con Thread

Thread th = new Thread(task);
th.setDaemon(true);
th.start();

  • Con ExecutorService:

ExecutorService.submit(task);

A parte de sobreexcribir el método call(), se pueden redefinir los métodos:

  • succeeded()

@Override protected void succeeded() {
    super.succeeded();
    updateMessage("Done!");
}

  • cancelled()

@Override protected void cancelled() {
    super.cancelled();
    updateMessage("Cancelled!");
}

  • failed()

@Override protected void failed() {
    super.failed();
    updateMessage("Failed!");
}

9.3. Service

Permite definir tareas de larga duracion interaccionables, se puede: pausar, relanzar, etc.

Se deberá extender la clase Service, implementando el método createTask cuya mision es retornar un objeto de tipo Task que implemente la tarea de larga duracion.

class FirstLineService extends Service<String> {
    protected Task<String> createTask() {
        return new Task<String>() {
            protected String call() throws IOException, MalformedURLException {
                    return null;
            }
        };
    }
}

Para ejecutar el Service, basta con invocar el método start()

FirstLineService service = new FirstLineService();
service.start();

11.12. Propiedades

Todoas los shapes disponen de estas propiedades:

  • Stroke Type: Permite definir donde se pinta el borde, pudiendo tomar como valores: StrokeType.INSIDE, StrokeType.OUTSIDE o StrokeType.CENTERED.

  • Stroke Width: Permite definir el ancho del borde.

path.setStrokeWidth(3.0);

  • Fill: Permite definir el relleno del shape

path.setFill(COLOR.BLUE);

  • Stroke: Permite definir el color del borde.

path.setStroke(Color.RED)

  • Stroke Line Join: Permite definir el comportamiento de los vertices, pudiendo ser: StrokeLineJoin.BEVEL (visel), StrokeLineJoin.MITER (punta) o StrokeLineJoin.ROUND (redondeada).

path.setStrokeLineJoin(StrokeLineJoin.BEVEL);

  • Stroke Miter Limit: permite definir la distancia del punto interior del vertice y el exterior, realizando un recorte si es necesario.

path.setStrokeMiterLimit(4);

  • Stroke Line Cap: Permite definir el estilo al terminar lineas. Puede ser: StrokeLineCap.BUTT, StrokeLineCap.SQUARE o StrokeLineCap.ROUND.

line.setStrokeLineCap(StrokeLineCap.SQUARE);

  • Smooth: Permite difuminar el pixelado.

path.setSmooth(false);

11.13. Operaciones

Se pueden realizar operaciones sobre los shapes, a traves de metodos estaticos de la clase Shape

  • Union: Calcula el shape que representa la union de dos shape.

//Drawing Circle1 (x,y,r, fill)
Circle circle1 = new Circle(250.0f, 135.0f, 100.0f, Color.DARKSLATEBLUE);

//Drawing Circle2 (x,y,r, fill)
Circle circle2 = new Circle(350.0f, 135.0f, 100.0f, Color.BLUE);

//Performing union operation on the circle
Shape shape = Shape.union(circle1, circle2);
//Setting the fill color to the result
shape.setFill(Color.DARKSLATEBLUE);

  • Subtraction: Calcula el shape que representa quitar un shape de otro.

//Drawing Circle1 (x,y,r, fill)
Circle circle1 = new Circle(250.0f, 135.0f, 100.0f, Color.DARKSLATEBLUE);

//Drawing Circle2 (x,y,r, fill)
Circle circle2 = new Circle(350.0f, 135.0f, 100.0f, Color.BLUE);

//Performing union operation on the circle
Shape shape = Shape.subtract(circle1, circle2);
//Setting the fill color to the result
shape.setFill(Color.DARKSLATEBLUE);

  • Intersection: Cacula el shape con la parte comun de otros shape.

//Drawing Circle1 (x,y,r, fill)
Circle circle1 = new Circle(250.0f, 135.0f, 100.0f, Color.DARKSLATEBLUE);

//Drawing Circle2 (x,y,r, fill)
Circle circle2 = new Circle(350.0f, 135.0f, 100.0f, Color.BLUE);

//Performing union operation on the circle
Shape shape = Shape.intersect(circle1, circle2);
//Setting the fill color to the result
shape.setFill(Color.DARKSLATEBLUE);

12. Graficos 3D

Permite dibujar geometrias en los ejes XYZ.

La clase base es javafx.scene.shape.Shape3D

Las figuras graficas predefinidas son: Box, Cylinder, Sphere y MeshView.

Se pueden definir las siguientes propiedades:

  • CullFace: Que puede tener como valores CullFace.NONE, CullFace.FRONT y CullFace.BACK

box.setCullFace(CullFace.NONE);

  • DrawMode: Que puede tener como valores DrawMode.FILL y DrawMode.LINE

box.setDrawMode(DrawMode.FILL);

  • Material: Representa la textura a aplicar al shape. Definido por la clase Material, se dispone de una subclase PhongMaterial, con las siguientes caracteristicas:

    • bumpMap: Emplea una imagen.

String urlImage = this.getClass().getResource("/sample/logo.gif").toExternalForm();
PhongMaterial material = new PhongMaterial();
material.setBumpMap(new Image(urlImage));
cylinder.setMaterial(material);

  • diffuseMap: Emplea una imagen y la difumina para limpiar vertices.

String urlImage = this.getClass().getResource("/sample/logo.gif").toExternalForm();
PhongMaterial material = new PhongMaterial();
material.setDiffuseMap(new Image(urlImage));
cylinder.setMaterial(material);

  • selfIlluminationMap: Emplea una imagen y la iluminada.

String urlImage = this.getClass().getResource("/sample/logo.gif").toExternalForm();
PhongMaterial material = new PhongMaterial();
material.setSelfIlluminationMap(new Image(urlImage));
cylinder.setMaterial(material);

  • specularMap: Emplea una imagen y aplica un reflejo.

String urlImage = this.getClass().getResource("/sample/logo.gif").toExternalForm();
PhongMaterial material = new PhongMaterial();
material.setSpecularMap(new Image(urlImage));
cylinder.setMaterial(material);

  • diffuseColor: Emplea un color y lo difumina.

PhongMaterial material = new PhongMaterial();
material.setDiffuseColor(Color.BLANCHEDALMOND);
cylinder.setMaterial(material);

  • specularColor: Emplea un color y aplica un reflejo.

PhongMaterial material = new PhongMaterial();
material.setSpecularColor(Color.BLANCHEDALMOND);
cylinder.setMaterial(material);

  • specularPower: PErmite definir el reflejo.

PhongMaterial material = new PhongMaterial();

13. Multimedia

Actualmente se soportan los siguientes codecs:

  • Audio: MP3; AIFF containing uncompressed PCM; WAV containing uncompressed PCM; MPEG-4 multimedia container with Advanced Audio Coding (AAC) audio

  • Video: FLV containing VP6 video and MP3 audio; MPEG-4 multimedia container with H.264/AVC (Advanced Video Coding) video compression

Se proporcionan los siguientes objetos:

  • Media: Representa el medio a reproducir, tiene la ubicacion y metainformacion. Se puede cargar contenido:

    • Remoto

Media media = new Media(http://download.oracle.com/otndocs/products/javafx/oow2010-2.flv);

  • Local

Media media = new Media(this.getClass().getResource("/sample/Morat.mp3").toExternalForm());

  • MediaPlayer: Define todos los metodos y atributos para controlar la reproducción.

MediaPlayer mediaPlayer = new MediaPlayer(media);
mediaPlayer.setAutoPlay(true);

  • MediaView: Componente visual que permite controlar el MediaPlayer

MediaView mediaView = new MediaView(mediaPlayer);

  • MediaErrorEvent: Evento que se lanza cuando MediaPlayer encuentra un error.

13.1. Maetainformacion del Media

mp.getMedia().getDuration();

13.2. Controlar la reproduccion

Para controlar la reproduccion se hace uso de los métodos de la clase MediaPlayer.

  • getMedia(): Permite obtener el Media reproducido actualemnte.

mp.getMedia().getDuration();

  • getStatus(): Permite conocer el estado de la reproduccion. Puede tomar como valores: Status.UNKNOWN, Status.HALTED, Status.PAUSED, Status.READY, Status.STOPPED.

Status status = mp.getStatus();

  • getStartTime(): Permite conocer el momento inicial del Media.

mp.getStartTime();

  • getCurrentTime(): PErmite obtener el actual tiempo de reproduccion.

mp.getCurrentTime();

  • seek(): Permite avanzar la reproduccion.

mp.seek(mp.getStartTime());

  • play(): Permite iniciar la reproduccion.

mp.play();

  • pause(): Pausa la reproduccion.

mp.pause();

  • setAutoPlay(): Inicia la reproduccion automaticamente.

mp.setAutoPlay(true);

  • setVolume(): Permite establecer el volumne de reproduccion.

mp.setVolume(20);
